Lease safety guarantees This design is safe because it enforces all of these: 1. one running step per agent at a time 2. each running step must pass acelogicExecutionGuard 3. each running step must bind a valid lease 4. each step is atomically claimed before execution 5. failures only release that step claim, not arbitrary others 6. the envelope remains source of truth for all running work 10. When concurrency is allowed Concurrency is valid only when: steps have satisfied dependencies steps belong to different agents steps are not human approval steps are claimable steps have valid per-agent identity + lease 12. When concurrency is NOT allowed Do not run in parallel when: two steps use the same assigned_agent_id a step depends on another incomplete step a human approval step is reached the agent lease cannot be acquired identity verification fails 13.
